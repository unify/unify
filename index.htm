Native-like applications for smartphones, tablets and desktops

Unify was built to improve the development efficiency of apps for smartphones. Currently it supports smartphones based on the platforms iOS, Android and WebOS. There is support for deskop operating systems like Windows or Mac OS as well. Support for tablet devices is planned for the next year.

The software makes use of a whole range of modern web technologies such as HTML5, CSS3 and JavaScript. A typical end user should not be able to distinguish an application built with Unify from a natively programmed application. Unify bypasses the traditional limitations of the web browser. Detecting the current geo location, accessing the file system, etc. is easy to implement.

Unify is made available under a dual license: MIT + Apache, Version 2.0

FAQ

Q: Why Unify?
A: Unify allows the programming of smart phone apps, regardless of the operating system. With Unify one is able to develop an application for the iPhone or Android-based phones like the HTC Desire in one process. This saves time and money.

Q: What's Unify?
A: Unify allows the programming of applications for smartphones, tablets and desktops with a single technology. The main claim of Unify is to allow solutions which the end user can not differentiate from a natively programmed application.

Q: What platforms are supported by Unify?
A: At the moment Unify supports smartphones based on Android, iOS and WebOS. It also supports the three desktop operating systems Windows, Mac OS and Linux.

Q: What about smartphones from Nokia, Blackberry, etc.?
A: The current priority is to support smartphone platforms with a working application market. In this aspect the iPhone and Android-based devices are currently the most interesting platforms. As soon as manufacturers of the other platforms achieve a similar popularity, Unify will also strive to support them. Support for BlackBerry OS 6.0, Windows Phone and Nokia's operating systems is planned for next year.

Q: Is not it difficult to desktop and smartphones using the same to develop technology?
A: The technical foundation is identical in all developments but large parts of the user interfaces will be optimized on a device category. Currently Unify distinguishes between smartphones (with touch screen), tablets and desktop computers. In contrast to the interface, the implementation of the business logic and data models is shared in all device categories.

Q: Why does Deutsche Telekom develop such a solution?
A: Currently there are no comprehensive and well-established solutions available. The cost of the native development for multiple devices are high. Each platform must be individually developed, maintained and improved.

Q: Do external partners contribute to Unify?
A: Unify has just been published as open source and thus allows external partners to use, extend and improve it was well. As open source software allows everybody to have a look at the code there is a good chance that the overall quality is higher than in closed software.

Q: What technology is Unify based on?
A: Unify makes use of several existing frameworks and technologies. At its core it is based on web technologies. Unify makes use of HTML5 and CSS 3. The whole UI is generated via JavaScript on base of the qooxdoo framework.

Q: Are web technologies powerful enough?
A: Web technologies are currently improved at a rapid pace. Browsers such as Firefox from Mozilla or Chrome from Google are improving their performance by about 30% per quarter. Impressive animations, playing audio and video files, accessing databases etc. is pretty easy to implement.

Q: What distinguishes Unify from other solutions based on web technology?
Unlike some older and less sophisticated solutions Unify aims to bring a native look&feel to applications. The usage of a professional and mature JavaScript framework helps developers building applications rapidly.

Q: Is Unify prepared for the future?
A: In contrast to technologies which are based on a translation of program code, Unify is future safe, as the foundation is built on open and widely distributed technology. Though the improvements in the underlaying software the speed and the possibilities are quite good even on older hardware.

Q: Could web applications integrate themselves to the system?
A: Web applications typically run in a browser, which does not allow any access to the system. That is the so-called sandbox. A typical web application has no way to read e-mails or files of the user. Technologies like Adobe AIR or PhoneGap allows the break the boundaries of these sandboxes. These are basically installed like native applications and allow the web applications inside them to enjoy the same level of access and integration.

Q: Why does Unify apply on PhoneGap?
A: PhoneGap is currently the only technology of its kind, which makes it possible to publish applications in the Apple AppStore and Android Market. For the moment this is the reason why it is more interesting for application developers than initiatives like BONDI or JIL. PhoneGap is open source and can thus be adapted perfectly to the needs of Unify.
